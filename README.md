# ♻️ ELV 廢塑膠產銷履歷示範平台

## 🎯 專案目標
建立廢塑膠（來源自報廢車輛）從出廠到產品銷售的完整履歷追蹤系統，透過QR碼掃描登錄提升可追溯性，作為未來歐盟ELV廢塑膠再利用政策接軌的示範平台。

## 🔄 作業流程
1. **產生QR碼** - 稽核團體為每批廢塑膠產生唯一QR碼
2. **階段登錄** - 各節點人員（出廠→運輸→處理→再製→銷售）掃描QR碼登錄資料
3. **履歷追蹤** - 完整記錄時間、重量、處理地點、產品型號等資訊
4. **資料管理** - 支援查詢、匯出，資料保存至少五年

## 🇪🇺 9.2.2 再生料溯源欄位(對接歐盟 ELV 新規)

登錄表單採 **stage 驅動**:選擇處理階段後只顯示該節點要填的少數欄位,其餘收進
「更多欄位」expander,降低現場填寫負擔。資料層保留全部欄位。新增 5 個關鍵欄位:

| 欄位 | 說明 |
|------|------|
| `recycled_ratio` | 再生料比率 %。**出廠** = 再生料來源純度;**後端段** = 成品再生含量(對歐盟 15%/25% 門檻) |
| `data_tier` | 資料可信度:實測(初級)/ 推估(二級)。質量平衡主數字會標示,推估值另加警示 |
| `is_elv_closedloop` | 是否報廢車閉環來源:是/否/未知 |
| `rrms_doc_id` | RRMS／報廢系統申報單號 |
| `material_type` | 物料類別:PP/PE/PS/PVC/ABS/其他 |

**質量平衡**採「節點取值」:進料=出廠重量、產出=再生/製造成品重量、損耗=進料−產出、
回收率=產出÷進料(運輸階段不計,避免重複計)。

## 🔐 帳號設定

不設定也能跑(自動進入**展示模式**並使用展示帳號,UI 會明確標示)。**正式對外請務必設定帳號**:

1. 複製 `.secrets_example.toml` 的內容到 `.streamlit/secrets.toml`(此檔已被 git 忽略),填入 `[auth]` 三個角色的密碼;
2. Streamlit Cloud 則貼到 App → Settings → Secrets;
3. 或改用環境變數 `APP_ADMIN_PW` / `APP_OPERATOR_PW` / `APP_VIEWER_PW`。

> 本專案不在程式碼或文件中存放真實密碼。詳見 `SECURITY.md`。
